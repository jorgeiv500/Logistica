1. Asignacion lineal¶

Contexto del Problema:

Una empresa de distribucin maneja productos que deben ser enviados desde sus almacenes hacia distintas tiendas en diferentes ciudades. La empresa busca minimizar los costos de transporte asegurando que cada tienda reciba los productos desde el almacn ms cercano o ms conveniente econmicamente.

Supongamos que la empresa tiene 3 almacenes (W1, W2, W3) y 4 tiendas (T1, T2, T3, T4). Los costos de transporte por unidad de producto de cada almacn a cada tienda son los siguientes:

T1 T2 T3 T4
W1 8 6 10 9
W2 9 12 13 7
W3 14 9 16 5

Cada tienda tiene una demanda especfica de unidades de producto, y cada almacn tiene una cantidad limitada de unidades disponibles para enviar. El objetivo es asignar de qu almacn debe surtirse cada tienda para minimizar el costo total de transporte, cumpliendo con las demandas de las tiendas y no excediendo la capacidad de los almacenes.

Formulacin del Modelo de Asignacin:

Definamos las variables y parmetros:

  • x_ij: nmero de unidades de productos enviadas del almacn i a la tienda j.
  • c_ij: costo de transporte por unidad del almacn i a la tienda j.
  • d_j: demanda de la tienda j.
  • s_i: suministro disponible en el almacn i.

El modelo busca minimizar la funcin objetivo, que es el costo total de transporte:

$ \text{Minimizar} \sum_{i,j} c_{ij} x_{ij} $

Sujeto a las siguientes restricciones:

  1. Satisfaccin de la Demanda: Cada tienda debe recibir su demanda exacta desde los almacenes. $ \sum_{i} x_{ij} = d_j \quad \forall j $

  2. Restricciones de Suministro: Cada almacn no puede enviar ms productos de los que tiene disponibles. $ \sum_{j} x_{ij} \leq s_i \quad \forall i $

  3. No Negatividad y Enteros: Las unidades enviadas deben ser nmeros enteros y no pueden ser negativas. $ x_{ij} \geq 0 \quad \forall i,j $ Y, si se requiere que las asignaciones sean en unidades enteras, entonces: $ x_{ij} \in \mathbb{Z}^+ \cup \{0\} \quad \forall i,j $

Este modelo de asignacin ayudar a la empresa a determinar la manera ms eficiente de distribuir sus productos desde los almacenes hasta las tiendas, optimizando los costos de transporte mientras se satisfacen las demandas de las tiendas y las limitaciones de suministro de los almacenes.

from pyomo.environ import *
import os

# Definir almacenes y tiendas
almacenes = ['W1', 'W2', 'W3']
tiendas = ['T1', 'T2', 'T3', 'T4']

# Costos de transporte por unidad desde cada almacn a cada tienda
costos_transporte = {
    ('W1', 'T1'): 8, ('W1', 'T2'): 6, ('W1', 'T3'): 10, ('W1', 'T4'): 9,
    ('W2', 'T1'): 9, ('W2', 'T2'): 12, ('W2', 'T3'): 13, ('W2', 'T4'): 7,
    ('W3', 'T1'): 14, ('W3', 'T2'): 9, ('W3', 'T3'): 16, ('W3', 'T4'): 5
}

# Demanda de cada tienda
demanda = {'T1': 20, 'T2': 15, 'T3': 30, 'T4': 25}

# Suministro disponible en cada almacn
suministro = {'W1': 40, 'W2': 60, 'W3': 50}

# Configurar el correo electrnico para NEOS
os.environ['NEOS_EMAIL'] = 'jorgei.romerog@utadeo.edu.co'

# Modelo
modelo = ConcreteModel()

# Variables de decisin: unidades enviadas desde almacn i a tienda j
modelo.x = Var([(i, j) for i in almacenes for j in tiendas], domain=NonNegativeIntegers)

# Funcin objetivo: Minimizar el costo total de transporte
modelo.cost = Objective(expr=sum(costos_transporte[i, j] * modelo.x[i, j] for i in almacenes for j in tiendas), sense=minimize)

# Restricciones de demanda
modelo.demanda = ConstraintList()
for j in tiendas:
    modelo.demanda.add(sum(modelo.x[i, j] for i in almacenes) == demanda[j])

# Restricciones de suministro
modelo.suministro = ConstraintList()
for i in almacenes:
    modelo.suministro.add(sum(modelo.x[i, j] for j in tiendas) <= suministro[i])

# Resolver el modelo usando NEOS
solver_manager = SolverManagerFactory('neos')
resultados = solver_manager.solve(modelo, opt='cplex')

# Imprimir los resultados
print("Costo total de transporte mnimo:", modelo.cost())

for i in almacenes:
    for j in tiendas:
        if modelo.x[i, j].value > 0:
            print(f"Del almacn {i} a la tienda {j}: {modelo.x[i, j].value} unidades")

2. TSP¶

Problema: Optimización de la Ruta de Viaje Entre Ciudades¶

Consideramos un conjunto de cuatro ciudades etiquetadas como A, B, C, y D. Nos enfrentamos al desafo de planificar un viaje que comience en una ciudad de origen, visite cada una de las otras ciudades exactamente una vez y luego regrese a la ciudad de origen, minimizando el costo total del viaje. Los costos de viaje entre cada par de ciudades son simtricos y se presentan en la siguiente tabla:

A B C D
A - 5 6 4
B 5 - 3 2
C 6 3 - 5
D 4 2 5 -

Objetivo:¶

El objetivo es identificar la ruta ms eficiente que cumpla con los requisitos anteriores, asegurando el menor costo de viaje total.

Formulación del Modelo MTZ:¶

Para abordar este problema, empleamos la formulacin MTZ, una tcnica de programacin lineal entera que nos ayuda a encontrar la solucin ptima. Definimos las siguientes variables y restricciones:

  • Variables:

    • (x_{ij}): variable binaria que toma el valor de 1 si se decide viajar de la ciudad (i) a la ciudad (j), y 0 en caso contrario.
    • (u_i): variable entera que representa el orden en el que se visita la ciudad (i), ayudando a evitar ciclos subptimos o subtours.

  • Restricciones:

    1. Unicidad de Salida y Llegada: Para asegurar que desde y hacia cada ciudad se viaje exactamente una vez, para todas las ciudades (i \neq j): [x{ij} + x{ji} = 1]
    2. Restricciones de MTZ para Evitar Subtours: Estas restricciones aseguran que no se formen ciclos que no incluyan a todas las ciudades, facilitando la visita a cada ciudad una nica vez antes de regresar a la ciudad de origen. Para cada par de ciudades (i) y (j), donde (i \neq j) y (N) es el nmero total de ciudades: [u_i - uj + N \cdot x{ij} \leq N-1]

  • Dominio de las Variables:

    • Cada (u_i) debe ser un entero positivo dentro del conjunto ({1, 2, 3, \ldots, N}).
    • Cada (x_{ij}) es una variable binaria, es decir, ({0, 1}).

Solución:¶

La aplicacin de este modelo nos permite calcular la ruta ms corta que cumple con el requisito de visitar todas las ciudades una vez, optimizando el costo total del viaje. Utilizamos herramientas de programacin lineal entera para resolver este modelo y obtener la ruta ptima.

    ('A', 'B'): 5, ('A', 'C'): 6, ('A', 'D'): 4,

import random

import math




# Datos del problema

distances = {

    ('A', 'B'): 5, ('A', 'C'): 6, ('A', 'D'): 4,

    ('B', 'A'): 5, ('B', 'C'): 3, ('B', 'D'): 2,

    ('C', 'A'): 6, ('C', 'B'): 3, ('C', 'D'): 5,

    ('D', 'A'): 4, ('D', 'B'): 2, ('D', 'C'): 5

}




# Definicin de la clase

class TSPProblem(Annealer):




    def move(self):

        """Intercambia dos ciudades en la ruta."""

        a = random.randint(0, len(self.state) - 1)

        b = random.randint(0, len(self.state) - 1)

        self.state[a], self.state[b] = self.state[b], self.state[a]




    def energy(self):

        """Calcula la longitud total de la ruta."""

        e = 0

        for i in range(len(self.state)):

            e += distances[(self.state[i-1], self.state[i])]

        return e


# Crear el problema e inicializar el estado

init_state = list('ABCD')

random.shuffle(init_state)

problem = TSPProblem(init_state)

# Intentar resolver el problema mediante recocido simulado

problem.set_schedule(problem.auto(minutes=1)) # set the schedule (temperature)
state, e = problem.anneal()

# Imprimir la solucin

print("%s: %i" % (state, e))
